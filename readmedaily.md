# Daily Logs - Reservation System Modernization (2026-04-24)

## 1. เราทำอะไร (Project Overview)
- แปลงระบบจองห้องประชุมแบบเดิมให้เป็น **Modern Soft UI** ที่สวยงามและนุ่มนวลขึ้น
- แยกโครงสร้างไฟล์เป็น `index.html`, `styles.css`, `scripts.js`, และ `api.js` เพื่อความสะดวกในการจัดการ
- เปลี่ยนระบบหลังบ้านจากการใช้ `google.script.run` มาเป็น **REST API (fetch)** เพื่อให้รองรับการรันบน GitHub Pages ได้

## 2. เราเพิ่มอะไร (Added Features)
- **Tabbed Interface**: สลับระหว่างหน้า "จองห้องประชุม" และ "รายการการจอง" ได้ทันที
- **Custom Dropdowns**: พัฒนาเมนูเลือกเวลาและเดือนเอง เพื่อแก้ปัญหาเมนูเดิมล้นหน้าจอ (Fixed Max-Height & Internal Scroll)
- **ID-based Mapping**: ระบบจำค่าห้องเป็น ID (เช่น ROOM_A) และแปลงเป็นชื่อไทยในหน้าจอ ช่วยให้แก้ไขชื่อห้องภายหลังได้โดยข้อมูลไม่พัง
- **Advanced Filtering**: ระบบกรองรายการจองตามเดือน/ปี ที่ฉลาดขึ้น (แสดงครบทั้งปีนี้และปีหน้า)
- **Extended Hours**: ขยายเวลาจองได้ถึง **20:00 น.**
- **Conflict Disclosure**: เมื่อจองทับกัน ระบบจะแจ้งชื่อทันทีว่าใครเป็นคนจองช่วงเวลานั้นไว้
- **Optional Phone Number**: ปรับให้ช่องเบอร์โทรศัพท์เป็น "ไม่บังคับ" เพื่อความสะดวกของพนักงานที่จองบ่อย

## 3. เราแก้อะไร (Improvements)
- **Performance Boost**: ลดการเรียก API จาก 2 รอบเหลือเพียง 1 รอบ และย้ายการคำนวณเวลาว่างมาไว้ฝั่ง Client ทำให้ระบบตอบสนองไวขึ้นมาก
- **Leading Zero Fix**: บังคับให้ Google Sheets เก็บเบอร์โทรเป็นข้อความ (Text) โดยการเติม `'` นำหน้า เพื่อไม่ให้เลข 0 หาย
- **Mobile Responsive**: ปรับแต่ง CSS ให้รองรับหน้าจอมือถืออย่างสมบูรณ์ และใช้การแสดงผลแบบ Card แทนตารางเพื่อให้ Gen X/Baby Boomer ดูง่าย

## 4. พบปัญหาอะไร และแก้ยังไง (Troubleshooting)
- **Dropdown Overflow**: เมนูเลือกเวลามันยาวทะลุหน้าจอและ Taskbar
  - *วิธีแก้*: สร้าง Custom Select Component เอง และจำกัด `max-height: 250px` พร้อมแถบเลื่อนข้างใน
- **ReferenceError (Modal Functions)**: พบปัญหาฟังก์ชันแจ้งเตือนหายไปขณะอัปเดตโค้ด
  - *วิธีแก้*: กู้คืนฟังก์ชัน `showStatusModal` และ `showConfirmModal` กลับสู่ `scripts.js`
- **Scrolling Issue**: หน้าจอบางส่วนเลื่อนไม่ได้บนมือถือ
  - *วิธีแก้*: ปรับ `align-items: flex-start` และเพิ่ม `padding-bottom` ให้เยอะขึ้น
- **Year Transition**: ปัญหาการมองเห็นข้อมูลข้ามปีในช่วงรอยต่อปีใหม่
  - *วิธีแก้*: ปรับตรรกะตัวกรองให้แสดงเดือนครบทั้งปีปัจจุบันและปีหน้าเสมอ (24 เดือน)

## 5. ประโยชน์ที่ได้รับ (Key Outcomes)
- **UX ดีขึ้น**: ผู้ใช้งานไม่ต้องรอโหลด "กำลังโหลด..." นานๆ ในช่องเวลาสิ้นสุด เพราะข้อมูลถูกดึงมาล่วงหน้าแล้ว
- **Error ลดลง**: มี Modal ยืนยันการยกเลิก เพื่อป้องกันคนเผลอกดผิด
- **Data Integrity**: เบอร์โทรศัพท์และชื่อห้องมีความถูกต้องและแม่นยำมากขึ้นในฐานข้อมูล
